01 About · 7년의 일관된 근육

복잡한 전문 내용을

방산 엔지니어 → 대면 컨설팅 → AI Native 빌더 — 세 번의 도메인 전환 동안 제가 매번 했던 일은 같습니다. 서로 다른 언어를 쓰는 이해관계자를 문서와 대화로 연결하는 작업입니다. 이 근육이 디피니트의 사업개발 매니저 역할에 직접 쓰일 수 있다고 판단해 지원드립니다.

2014.03 — 2017.06
3년 4개월

정부 · 기관 보고 문서가 체화된 시기

한화오션(주) · 특수선무장설계팀 · 통신/무장 기본설계

장보고-Ⅲ 잠수함 프로젝트에서 국방과학연구소를 포함한 외부 기관 보고용 자료를 팀 내 담당자들로부터 취합·정리·작성하는 역할을 3년 4개월간 담당했습니다. 외부 기관 문서는 수치 하나, 용어 하나 틀리면 끝이라는 긴장 아래에서 일했고, 이때 체화된 정확성·검증 중심의 태도는 지금도 작동하는 근력입니다.

동시에 잠수함 부품 조달 이슈를 독일 제조사와 영어 핫라인으로 직접 조율하며, 복수 이해관계자(독일 제조사·국내 조달부서·내부 설계 부서) 사이에서 문제를 구조화해 문서로 정리하는 훈련을 받았습니다. 디피니트 공고의 "고객 앞에서 기술을 비즈니스 언어로 설명할 수 있는 분" 요건은, 이 시기에 시작된 작업 방식의 자연스러운 연장선입니다.

2022.04 — 현재
4년+

수백 명 의사결정자를
언어로 움직인 4년

대치동 정시 컨설팅 · 프리랜서 컨설턴트

매년 바뀌는 입시제도라는 복잡한 규칙 체계를 매 시즌 수백 명의 학생·학부모를 대면해 비전문가 언어로 옮기는 작업을 4년간 반복했습니다. 사전 설문 → 대면 상담 → 후속 관리 → 파이널 콜로 이어지는 고밀도 사이클을 혼자 설계·운영했고, 결과는 재등록률과 지인 추천 리텐션으로 검증되었습니다.

이 사이클은 B2B 영업 사이클(리드 분석 → 제안 → 협상 → 클로징 → 리텐션)과 구조적으로 동일합니다. 의사결정자에게 복잡한 정보를 전달하고, 결정을 끌어내고, 사후 관계를 유지하는 작업은 도메인은 달라도 본질이 같습니다. 디피니트 공고의 "귀사의 이 문제를 이렇게 줄일 수 있습니다"로 말할 수 있는 근육이 이 4년 동안 가장 두껍게 쌓였습니다.

AI를 작업 방식으로
체화한 시기

前 에듀테크 스타트업 운영PM · 현재 프리랜서 AI 빌더

Claude Opus 4.7 · Codex · ChatGPT · Gemini를 코딩 파트너로 병렬 운용하며, 최근 몇 달 사이에 세 개의 AI 프로덕트(영상 자동화 앱 · SNS 트렌드 수집 앱 · 실시간 매칭 웹앱)를 기획부터 배포까지 단독으로 완결했고, 그중 실시간 매칭 웹앱은 현재도 유지보수 중입니다.

이 과정에서 익힌 것은 단순한 코딩이 아니라 "AI 초안 + 사람의 판단" 구조의 운용입니다. 작업 단계별로 모델을 분업시키고, 한 모델의 출력을 다른 모델로 교차 검증하는 워크플로가 코드 작업뿐 아니라 문서·리서치·분석 작업 전반에 그대로 적용됩니다. 디피니트 공고의 "AI 활용은 우대가 아니라 이 역할의 기본 작업 방식"이라는 문장이, 제가 매일 일하는 방식을 그대로 묘사한 것이라고 느꼈습니다.

디피니트 공고 직접 대응 강점
  • AI 활용 = 기본 작업 방식. ChatGPT · Claude · Gemini · Perplexity 일상 운용. 작업 단계별 모델 분업 + 교차 검증 구조 체계화.
  • 고위험 · 고격식 문서 경험. 한화오션 국방과학연구소 보고 3년 4개월 + 대치동 의사결정자 상대 4년 + 프리랜서 PRD · 설계 판단 문서.
  • 기술을 비즈니스 언어로 번역. 본 페이지 02번 섹션의 "A사 AI 도입 제안서 샘플"이 직접 증거. 같은 구조가 정부지원사업 사업계획서로도 매핑 가능.
  • 제조업 현장 감각. 한화오션 방산 제조 3년 4개월 — DARVIS가 풀려는 "감에 의존한 판단"이 어떻게 발생하는지 현장에서 본 경험.

04 Domain · 제조업 현장 감각

DARVIS가 풀려는 문제를
현장에서 본 경험이 있습니다

디피니트 우대사항의 "제조업 도메인 경험"에 대해, 한화오션 특수선무장설계팀에서 방산 제조(잠수함 프로젝트) 현장의 운영 문제를 3년 4개월간 직접 겪었습니다. 일반 제조업과는 차이가 있지만, 부서 간 데이터 단절·다자 이해관계 조율·외부 제조사 품질 이슈 같은 구조적 문제는 같습니다.

2014.03 — 2017.06 · 한화오션(주) 특수선무장설계팀

장보고-Ⅲ 잠수함 프로젝트 · 3년 4개월

Pressure Hull Penetrator 장비 담당. 독일 제조사와 영어 핫라인을 유지하며, 국내 조달부서와 독일 제조사 사이에서 생산 설계 과정 중 발생하는 이슈를 조율했습니다. 여러 시스템 · 여러 부서 · 여러 국가의 데이터가 단절된 환경에서 문제를 정의하고 해결하는 작업이 일상이었습니다.

전투정보실(CIC) 배치설계. Aveva Marine으로 잠수함 전투정보실 내 장비 배치설계 일부를 담당했고, 배관설계부서 · 전장설계부서와의 긴밀한 협업이 필수였습니다. 각 부서가 같은 문제를 "불량", "스펙 불일치", "납기 리스크"로 다르게 부르는 환경에서, 같은 문제를 정확한 언어로 통일하는 작업을 반복했습니다.

국방과학연구소 보고 자료 작성. 부서 내 업무별 담당자들로부터 자료를 취합해 외부 기관(국방과학연구소 등) 보고용 자료를 정리하는 역할을 담당했습니다. "감이 아닌 데이터로, 문제의 원인과 조치를 보여드립니다"는 DARVIS의 메시지가 익숙하게 느껴지는 이유는, 그 반대편(감에 의존한 판단으로 같은 문제가 반복되는 환경)을 현장에서 보았기 때문입니다.
